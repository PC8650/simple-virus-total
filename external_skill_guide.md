# 外部技能库 (External Skill) 功能说明

## 1. 功能概述

本项目支持加载外部 Markdown 技能文件，在 AI 分析 VirusTotal 数据时，将匹配到的技能文件内容作为补充提示词注入上下文。

**核心机制**：启动时扫描指定目录下的 `.md` 文件，提取 YAML Frontmatter 中的 `tags` 列表，在内存中建立 `tag → skill文件` 的倒排索引。当 VirusTotal 返回的 JSON 数据中命中某个 tag 时，系统会统计每个 Skill 文件命中的标签总数，并将**匹配度最高（命中标签最多）**的 Top N 个技能文件内容拼接到 AI 提示词末尾。

## 2. 配置方式

在 `application.yml` 中配置：

```yaml
external-skill:
  enable: true                    # 是否启用外部技能库
  dir: "D:/my-security-skills/"   # 技能文件所在目录的绝对路径
  top-limit: 3                    # 每次分析最多注入的技能文件数量
```

| 参数 | 说明 |
|------|------|
| `enable` | 总开关。设为 `false` 时完全跳过扫描和匹配 |
| `dir` | 系统启动时递归扫描该目录下所有 `.md` 文件 |
| `top-limit` | 控制单次注入的文件数量上限。**该值越大，注入的外部文本越多，AI 遗漏原始数据的风险越高** |

## 3. Skill 文件格式规范

系统仅识别包含标准 YAML Frontmatter 的 `.md` 文件。不符合格式的文件会被静默跳过。

```markdown
---
description: "分析 PE 文件中的 Overlay 附加数据"
tags:
  - overlay
  - peexe
  - T1027
---

## 分析步骤
1. 检查文件末尾的熵值...
```

**格式要求**：
- 文件必须以 `---` 开头，YAML 块以 `---` 结尾
- 必须包含 `tags` 列表，这是触发匹配的唯一依据
- `description` 为可选字段
- 如果主文件同级存在 `references/` 目录，其中的 `.md` 文件将作为补充材料一并注入

## 4. 匹配与排序逻辑

本项目采用**“等权重特征碰撞”**算法。系统不预设标签的优先级，而是通过计算“匹配密度”来决定优先级：

1. **精准匹配**：系统使用正则表达式词边界（`\b`）对 VirusTotal 的原始 JSON 进行扫描。只有当标签作为一个独立的单词出现时（例如 `ip` 不会匹配到 `zip`），才计为一次命中。
2. **分值累加**：每一个命中的标签为该 Skill 文件贡献 **1 分**。
3. **优胜劣汰**：如果一个 Skill 文件定义了多个标签（如 `T1055`, `injection`, `peexe`），且这三个特征在 VT 数据中同时出现，该文件将获得 3 分。
4. **最终排序**：系统按总分从高到低排序，取前 `top-limit` 个文件。

**逻辑优势**：这种机制会自动让那些“描述更精准、标签更丰富”的专业技能文件脱颖而出，而只有单个宽泛标签（如仅有 `file`）的通用指南则会排在末尾。

## 5. 已知局限与负面效应

使用外部技能库时，需要了解以下约束：

### 5.1 注意力稀释
外部技能文件的内容会被追加到 AI 提示词末尾。**注入的外部文本越多，AI 对原始 VirusTotal JSON 数据的关注度越低**，可能导致：
- 遗漏 JSON 中的关键字段（如 `behaviour` 数组中的沙箱数据）
- 对原始数据的引用不够精确
- 输出中过度依赖外部指南的描述，而非基于实际数据的分析

**建议**：`top-limit` 不宜超过 3。

### 5.2 标签匹配的精度限制
本项目采用基于正则词边界的纯文本匹配（`\b` + `Pattern.quote`），不具备语义理解能力。这意味着：
- 无法识别同义词（如 tag 为 `process injection`，但 JSON 中写的是 `code injection`）
- 标签的质量完全取决于 Skill 文件的编写者。如果 Skill 文件只打了一个极其罕见的标签，可能永远无法被触发。

### 5.3 不具备代码执行能力
本项目是纯文本推理辅助工具。外部 Skill 目录中可能包含的 `.py`、`.sh` 等脚本文件会被忽略，系统仅读取 `.md` 文件。AI 只能阅读指南中的文字描述进行推理，不会执行任何代码。

### 5.4 启动扫描开销
目录中文件数量过大时（如数百个），启动阶段的递归扫描和 YAML 解析会占用额外时间。所有索引数据常驻内存，文件数量极多时会增加内存占用。

## 6. 关于 Skill 文件格式标准

本项目的 Skill 解析逻辑（YAML Frontmatter + tags 列表 + references 目录）参照了开源项目 [Anthropic-Cybersecurity-Skills](https://github.com/mukul975/Anthropic-Cybersecurity-Skills) 的文件组织规范。

该项目由社区维护者基于 Anthropic 的安全研究方向创建，包含约 700+ 个网络安全分析 Skill 文件，涵盖 MITRE ATT&CK 战术技术、恶意软件家族分析、漏洞利用检测等场景。每个 Skill 采用统一的目录结构：

```
skill-name/
├── SKILL.md            # 主技能文件（含 YAML Frontmatter）
└── references/         # 可选的补充参考材料
    ├── ref1.md
    └── ref2.md
```

**本项目兼容该目录结构和文件格式**，但不对该项目的内容质量做任何担保或推荐。如需编写自定义 Skill 文件，建议参照上述格式标准组织目录和编写 YAML 头部，以确保本系统能够正确识别和加载。
